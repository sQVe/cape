README V2 — proposed structure

README.md — Cape

Cape

Opinionated Claude Code workflow — plan, build, and ship with consistent habits.

Quick start

  1. Install: claude plugin add sQVe/cape
  2. Open any repo and say "commit my changes" — cape's commit skill fires automatically.
  3. Say "review my diff" or "open a PR". Those are your first three skills.
  4. When you're ready to plan a feature, say "let's brainstorm X" to enter the PLAN chain.

The three chains

PLAN brainstorm → challenge → write-plan
BUILD execute-plan → tdd → commit
SHIP review → finish-epic → pr
How auto-triggering works: Cape listens to what you say via the don-cape router. Say "commit", "review", or "brainstorm" in natural language and the right skill fires. You can also invoke directly with /cape:skill-name. Skills marked with a dashed border in the chain diagrams fire automatically — you never call them manually.

Top 4 concrete changes

Change 01

Replace flat skill list with three named chains

README opens with PLAN → BUILD → SHIP. Each chain shows the skill order and which skills auto-fire. Newcomer immediately sees structure, not a 22-item dump.

PLAN: brainstorm → challenge → write-plan
BUILD: execute-plan → tdd → commit
SHIP: review → finish-epic → pr
Change 02

Declare a 3-skill starter set

Day 1 is just commit, review, pr — the top 3 by usage. Everything else is hidden until the newcomer asks "what else can I do?" Progressive disclosure beats documentation walls.

Change 03

Explain auto-triggering up front with an example

The README currently says nothing about don-cape. A one-paragraph callout — "say 'commit' and this fires" — builds trust before the magic happens unexpectedly.

Change 04

Retire 5 zero-use skills

analyze-tests, branch, design-an-interface, explain, find-test-gaps never fired in 10 days of real use. Each unused skill costs a newcomer attention. Remove or fold functionality into active skills.

Data source: 10 days of real Claude Code logs from the cape author. Skill fire counts: commit 29, tdd 11, review 12, pr 12, brainstorm 9, execute-plan 7, conform 7, challenge 7, write-plan 6. Five skills: 0 fires. The redesign follows the evidence, not intuition.
