Four findings that drove every redesign

  1. ~9 of 22 skills do the work. Five fired zero times — dead weight in the router and in context.
  2. The real shape is three chains, not a flat list:
    brainstorm→challenge→write-plan · execute-plan→expand-task→tdd→commit · finish-epic→pr (+conform).
  3. Trigger split: chained skills (tdd, execute-plan, challenge, finish-epic) fire automatically; commit/review/pr/conform are driven by /slash.
  4. conform is a post-commit afterthought, and the debug-issue→fix-bug chain is bypassed. 37 /clears mean work spans many short sessions — br is the only thing that persists.

Agents (the automation engine): codebase-investigator 30, test-runner 20, fact-checker 12, internet-researcher 10, code-reviewer 8 — then a cliff to test-auditor / notebox-researcher / bug-tracer at 2 each.

02 Six lenses on V2

① Workflow Architect state machine

"V1 is a bag of 22 skills routed by a lookup table; V2 is a three-chain state machine (Plan→Build→Ship + Bug track) where br epics carry all state across /clear."

  1. Delete the 5 dead skills — pure routing noise.
  2. conform becomes a mandatory gate inside Build (tdd→conform→commit), not a post-commit fixup.
  3. fix-bug requires a br bug — if none exists, route to debug-issue first.

② Minimalist subtractive

"Cut anything the data didn't touch; merge anything that only fires as a sub-step. 22 skills + 8 agents → 14 + 5 (≈37% removed)."

  1. Fold challenge into brainstorm (it never fires standalone).
  2. Inline expand-task into execute-plan (internal-only).
  3. Cut analyze-tests + find-test-gaps — TDD at 11 prevents gaps rather than hunting them.

③ DX & Ergonomics friction

"Eliminate remembering: convert manual quality gates into automatic chain steps and surface br state on every session start."

  1. conform runs before git commit — never typed again.
  2. session-start hook injects br ready --short so "continue" just works after a /clear.
  3. review folds into /pr as step 1 — impossible to open a PR without a review pass.

④ Information Architect taxonomy

"Replace the flat, mixed-axis bag with six namespaced lifecycle tracks, loaded progressively — ~85% less per-session context for single-track work."

  1. Track namespacing + verb naming: build:plan/run/close, fix:diagnose/patch, ship:commit/pr…
  2. Progressive disclosure: session start injects a ~200-token track manifest; load a track's skills on demand.
  3. Absorb the 5 dead skills into tracks or remove them.

⑤ Automation & Power-User enforcement

"Move invariants into hooks (PreToolUse DENY), keep judgment in skills, and make br the only persistence layer that survives /clear."

  1. Hard TDD block: PreToolUse DENY on Edit/Write when tdd-state isn't "test written".
  2. Blocking conform gate: DENY git commit while staged changes violate conventions.
  3. Cut bug-tracer + notebox-researcher into codebase-investigator --mode flags (8 agents → 5).

⑥ Newcomer Advocate learnability

"A newcomer fails because cape shows 22 alphabetical skills with no order and no explanation of auto-triggering."

  1. Lead with the three named chains (PLAN→BUILD→SHIP) showing order + what auto-fires.
  2. Day-1 starter set of 3 (commit, review, pr) — hide the rest until asked.
  3. Retire the 5 dead skills — clutter is a learnability tax.

Persistence layer: br epics/tasks/bugs carry state across every /clear. A session-start hook surfaces the next ready item so "continue" always works.

Do now — unanimous / strong, low regret

  • Delete the 5 dead skills (analyze-tests, branch, design-an-interface, explain, find-test-gaps). All six agents. Zero fires in 10 days; they cost router noise and per-session context.
  • Restructure cape around the 3 chains + bug track, with br as the durable state machine. Unanimous. The chains already exist in practice — make them first-class instead of emergent.
  • Move conform into the commit gate (run before git commit, block on violations). Architect + DX + Automation. Today it's remembered post-hoc and generates fixup commits.
  • Fold review into /pr and challenge into brainstorm. Both only ever fire adjacent to their host step; folding removes two things to remember.
  • Session-start hook surfaces br ready; merge the 3 near-dead agents into codebase-investigator modes. Makes 37-clear reality painless and trims 8 agents → 5 with no capability loss.

Decide — genuine tensions to resolve

  • How hard to enforce? Automation wants PreToolUse DENY hard-blocks for TDD & conform; DX/Architect lean toward chain steps and softer gates. Trade-off: guaranteed invariants vs. flow interruption when the model is mid-task. Likely answer: hard-block conform at commit, soft-guide TDD.
  • How much taxonomy? Information Architect wants 6 namespaced tracks (build:/fix:/ship:…) + lazy loading; Minimalist wants fewer skills, flat. Namespacing aids legibility & context cost but adds ceremony to a ~14-skill set. Reconcile: namespace by the 3 chains, not 6 tracks.
  • Keep the low-frequency skills? refactor / debug-issue / task-refinement each fired ≤1. Low-frequency-but-valuable vs. dead. debug-issue earns its place as the bug-track entry; refactor & task-refinement need a usage verdict before cutting.
